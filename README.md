# 📄 PDF → LaTeX 转换工具（via DeepSeek）

一个 Python 脚本，调用 DeepSeek 大模型，将中文数学建模论文 PDF 自动转换为结构清晰的 LaTeX 文档。

## ✨ 能做什么

- 提取 PDF 中的全部文字（支持多页）
- 自动生成符合中文排版习惯的 LaTeX 文件（`ctexart`）
- 将原文中的公式转为 LaTeX 数学环境
- 表格转为三线表（`booktabs`）
- 保留图片、参考文献的占位符
- 本地运行，你的 PDF 不会上传到任何地方（仅将提取的文字发送给 DeepSeek API）

## ⚠️ 已知限制（使用前必读）

本项目是一个辅助工具，**并非完美的一键转换方案**，存在以下已知限制：

- **表格处理**：PDF 中的表格会被提取为纯文字，**无法自动识别和重建表格结构**。生成的三线表可能需要手动核对和调整数据。
- **图片处理**：所有图片均转为 `\includegraphics` **占位符**，不会提取或生成实际图片文件。你需要手动从原 PDF 中导出图片并插入。
- **字体还原**：本工具不会分析原文档的字体，LaTeX 输出使用 `ctexart` 默认字体（宋体/仿宋/黑体）。**会与原文档字体有差异**，可自行在导言区修改字体设置。
- **格式偏差**：生成结果会力求结构完整，但细节排版（如段落间距、列表样式）可能与原文不完全一致。

> 💡 **建议**：将输出视为一份“粗排草稿”，在此基础上手动精修，远比从零写起高效得多。

## 🔧 配置与使用

### 1. 获取 DeepSeek API Key

> 🔑 **API 获取技巧**：
> 1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册/登录。
> 2. 在左侧菜单找到 **API Keys**，点击“创建新的 API Key”。
> 3. **新用户通常有免费赠送额度**，足够你调试和转换几十篇论文。用完后按量付费，价格很低。
> 4. 复制生成的 `sk-` 开头的密钥并妥善保存（只显示一次，丢了要重新创建）。
> 5. 推荐使用 `deepseek-chat` 模型（V3），速度与成本平衡最佳；若需更强推理能力可选 `deepseek-reasoner`（R1）。

### 2. 设置环境变量

1. 将项目中的 `.env.example` 复制一份，重命名为 `.env`
2. 打开 `.env`，把内容修改为：

```env
DEEPSEEK_API_KEY=你的真实密钥
```

## 🔧 配置与使用

### 3. 准备 PDF 文件

将你要转换的 PDF 放在与脚本相同的目录下（或记住它的绝对路径）。

### 4. 修改脚本中的路径

打开 `pdf2latex.py`，修改以下变量为你自己的路径：

```python
PDF_PATH = "你的论文.pdf"          # PDF 文件路径
ENV_PATH = ".env"                 # 环境变量文件路径
OUTPUT_TEX_PATH = "output.tex"    # 输出 LaTeX 文件路径
```
### 5. 运行脚本

```bash
python pdf2latex.py
```
稍等片刻，生成的 .tex 文件就会出现在你指定的位置。

### 6. 编译 LaTeX 文件
📥 LaTeX 环境推荐：

- 本地安装：TeX Live（Windows/Linux/Mac 通用，免费完整发行版）
- 在线编辑：Overleaf（注册即用，无需安装，支持中文，团队协作方便）

打开 .tex 文件，用上述任一工具编译即可生成 PDF。

## 💡 使用技巧
- 先跑通再调优：首次使用建议用一篇 5 页以内的短论文测试，熟悉整个流程。
- 图片手动补全：脚本生成的 \includegraphics{...} 仅保留文件名占位符，你需要手动截图或导出原图，放入项目文件夹并在代码中指定文件名。
- 表格核对：PDF 中的三线表被提取为文字后，LaTeX 中会尝试重建结构，但数据错位是常见现象，务必逐表核对。
- 字体调整：如果对默认字体不满意，可在 .tex 文件导言区添加：

```latex
\setCJKmainfont{SimSun}   % 宋体（正文）
\setCJKsansfont{SimHei}   % 黑体（标题）
\setCJKmonofont{FangSong} % 仿宋（代码块/引用）
```
- 具体字体名取决于你的系统已安装字体。
- 长文档分批处理：论文超过 30 页建议拆分为多个 PDF 分批转换，避免超出 API 上下文窗口。
- 温度参数调低：代码中 temperature=0.2 已偏向确定性输出，若结果过于随意可降至 0.1。
